## Current version: 1.6.0
`fxmanifest.lua` — versioning policy: **MINOR** bumps per significant feature/redesign batch, **PATCH** for small fixes. Phase 1 completed at `1.0.0` (2026-07-09); Phase 2 features ship as `1.x.x`.

### Since 1.0.0 (Phase 2, in progress)
- **My Sim app** (mobile data plan + WiFi zone status) — Phase 1 of a planned dead-signal-zone system.
- **Full i18n** — every app's UI now follows Settings > Language, not just the shell/core screens; Lua-side notifications gained a per-recipient translate helper (`Utils.TFor`) so a push always renders in the RECIPIENT's own saved language.
- **Store app redesign** — sidebar category navigation (Finance/Business/Property/Social Media/Transportation).
- Bugfixes: battery-percentage notifications no longer show decimals; Service Station repair/restore/switch-OS now catches a silently-failed DB write instead of reporting success anyway; Cab's Decline button now actually clears the driver's request card.

## 1.0.0 — Phase 1 complete (2026-07-09)
Full baseline ships:
- Bridge system: framework (Qbox/QBCore/ESX), inventory (ox_inventory), voice (pma-voice), media (Fivemanage), banking (Renewed + fallback), garage (JG), housing (BCS), interaction (ox_target) — all config-driven with `custom` fallbacks.
- Device-centric model with auto-applied, idempotent-migration schema.
- Security: ownership checks, input validation, per-action rate limiting on every endpoint.
- Device setup + SIM/number generation mirrored into the framework's own player record.
- Account creation + lost-phone recovery (email/password/security question, plain-text by design).
- Contacts, Calls, Messages (incl. group chats), Gallery + Camera, Battery & charging (incl. powerbank + Service Station repair), **mBanking** (myBCA-inspired redesign), Garage, Property, Settings (native-OS-styled), custom apps.
- Phase 2-onward apps already shipped by this point: Job, Dark Chat, Economy Market, Mail, Birdie, Downtown Cab Co., GPS, Notepad, Calculator, Store, Advertise.
- Major UX work: walk-while-using, configurable emotes/props with a watchdog, centralized OS-differentiated design system, real app icon artwork, slide open/close/peek phone animation, Control Center, docked/floating push notifications with slide animation, lock-screen notification history.
- Offline test suite (`tests/run.lua`) + `install/ox_inventory.txt`.

## Operational notes for every release
- Keep `Config.Debug = false` for production.
- Rebuild `web/dist` before packaging (`cd web && npm run build`) — it's git-ignored.
- Verify database migrations applied cleanly after an update+restart (`server/migrations.lua` logs on startup).
- Run `lua5.4 tests/run.lua` before shipping.
- Prepare escrow packaging separately if selling through Cfx/Tebex — `config/*.lua` and a few other paths are deliberately `escrow_ignore`d (see [Architecture Notes](./architecture-notes)).
